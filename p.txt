Implement a Results & Skill Report upgrade for the existing English Quiz MVP.

Do NOT add a backend, login, database, payments, analytics SDKs, or external dependencies.
Do NOT rewrite the existing architecture.
Do NOT regress Vocabulary, Grammar, Mixed, Reading, Listening, feedback form, or static Cloudflare deployment.
Keep the app free, browser-only, and fast.

The product should feel more like a serious language assessment report.
It must remain honest: results are estimated and are not an official CEFR certificate.

==================================================
PRODUCT GOAL
==================================================

Upgrade the result screen so it shows:
- an overall estimated level when appropriate
- a skill-by-skill breakdown
- a short interpretation of strengths and weak areas
- a clear indication of which skills were tested and which were not
- a practical next-step recommendation

Use the spirit of CEFR self-assessment / skill reporting:
- separate receptive and knowledge skills where applicable
- show approximate results, not official certification
- support writing-style honest labels like:
  "Estimated Level"
  "Estimated Reading Level"
  "Estimated Listening Level"
  "Skill breakdown"
  "Strengths"
  "Needs practice"

Reference behavior inspiration only:
- CEFR self-assessment grids
- British Council / EnglishScore style skill reporting
- clear per-skill score presentation
Do not copy any proprietary wording.

==================================================
1. RESULT MODEL
==================================================

Extend result data safely so the UI knows:
- quiz type taken
- total questions answered
- correct count
- percentage
- estimated CEFR level
- per-skill breakdowns for:
  - Vocabulary
  - Grammar
  - Reading
  - Listening
- whether a skill was actually tested in this session
- any passage/clip context required for review views

If a skill was not tested in the current session, do not display fake numbers for it.

Suggested data shape:
- overallResult
- skillResults
- testedSkills
- untestedSkills
- confidence / sample-size note if needed

==================================================
2. RESULT SCREEN CONTENT
==================================================

The result screen should include these sections:

A. Main result card
- Large overall estimated level label
- Short 1–2 sentence explanation
- Score summary such as correct / total and percentage
- Keep the existing disclaimer that this is not an official CEFR exam

B. Skill breakdown
- Show a compact table or card grid of tested skills only
- Each tested skill should show:
  - skill name
  - correct / total
  - percentage
  - a short qualitative note:
    - Strong
    - Solid
    - Needs practice
- Use honest labeling:
  - Vocabulary-only sessions should not imply Reading/Listening were tested
  - Reading-only sessions should say "Estimated Reading Level"
  - Listening-only sessions should say "Estimated Listening Level"
  - Mixed sessions should show the skills actually included in the mixed session

C. Strengths and needs practice
- Based on the session data, display:
  - one or two strengths
  - one or two areas to improve
- Keep this concise and honest
- Do not invent diagnosis beyond the quiz scope

D. Recommended next step
- Show a simple suggestion such as:
  - "Next: practice grammar at the same level"
  - "Next: try Reading to confirm your placement"
  - "Next: review A2 vocabulary and grammar"
- Make this dependent on quiz type and score band

E. Review
- Keep the detailed review list
- Ensure it remains readable and compact

F. Feedback form
- Keep the existing feedback link below the primary action if present
- Do not let it visually dominate the result screen

==================================================
3. SCORING AND INTERPRETATION
==================================================

Implement a small interpretation layer:
- 0–39%: Needs a lot of practice
- 40–59%: Developing
- 60–79%: Solid
- 80–100%: Strong

Use these bands only as a lightweight qualitative guide, not as official language certification.

Map the bands to short supportive copy such as:
- Needs a lot of practice
- Developing foundation
- Solid control
- Strong performance

Rules:
- Do not claim official CEFR placement accuracy
- Do not overstate small sample sizes
- If the session is short, show a note like:
  "This result is based on a short practice set, so treat it as a directional estimate."
- If only one skill was tested, label the result as that skill-specific estimate rather than a broad overall level.

==================================================
4. UI DESIGN
==================================================

Create a polished but restrained result layout:
- keep the main result above the fold on mobile
- use compact cards / rows for skill breakdown
- use semantic HTML
- maintain clear hierarchy
- maintain 44px touch targets
- no decorative clutter
- no icon-only buttons unless accessible labels exist
- works at 375px and desktop
- use current style system and spacing tokens
- keep dark-mode readiness if already present
- respect prefers-reduced-motion

Result screen should never feel like a certificate or official exam report.
It should feel like a helpful practice summary.

==================================================
5. TESTED VS UNTESTED SKILLS
==================================================

Important:
- If Vocabulary only was tested, show only Vocabulary in the skill breakdown.
- If Grammar only was tested, show only Grammar.
- If Reading only was tested, show only Reading and label it clearly.
- If Listening only was tested, show only Listening and label it clearly.
- If Mixed is Vocabulary + Grammar, show both.
- Do not show Reading or Listening in Mixed unless they were actually tested.
- Untested skills may be mentioned in a small note like:
  "Reading and Listening were not part of this session."
