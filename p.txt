Fix ONLY the Listening timer blink bug in the current English Quiz MVP.

CURRENT BUG
In Listening mode, when the timer / playback status is shown or updated, the whole page or large parts of the screen blink/flicker.
Only the timer text or its tiny status indicator should visually update.
The entire page, card, or container must not blink.

IMPORTANT SCOPE
- Fix only the Listening timer/status visual flicker bug.
- Do NOT rewrite the app.
- Do NOT change quiz logic, scoring, audio gating, adaptive flow, or result logic.
- Do NOT add dependencies.
- Do NOT redesign the listening UI.
- Keep the current architecture and styling approach.

GOAL
Make timer/status updates visually stable.
If any attention effect is needed, it must apply only to the smallest possible timer/status element, not to parent containers.

DEBUG THE ACTUAL ROOT CAUSE
Inspect at minimum:
- src/ui/components/listening-player.ts
- src/ui/screens/quiz-screen.ts
- src/styles/main.css
- any utility/class affecting timer, status, pulse, blink, animation, opacity, transition, or rerender state

Check for these common causes:
1. A blink/pulse/animation class applied to a parent container instead of the timer text
2. CSS animation on `opacity`, `filter`, `transform`, or `box-shadow` attached to the whole player/card/screen
3. Re-render that replaces the whole listening player on every timer tick
4. State updates rebuilding the full quiz screen instead of only updating the timer/status node
5. CSS transitions on broad selectors causing layout-wide flicker
6. Animation class toggled on a wrapper that contains the whole question area
7. Status text being re-mounted each second instead of updated in place

FIX REQUIREMENTS

1. Scope the visual update correctly
- If the timer changes every second, update only the timer/status text node.
- Do not re-render the whole listening section on each tick if avoidable.
- If a CSS animation exists, attach it only to a very small element like:
  - `.listening-timer`
  - `.listening-status-dot`
  and never to:
  - page wrapper
  - screen wrapper
  - card wrapper
  - player wrapper
  - question container

2. Prefer stable text updates over blinking animation
- If the current blink effect is unnecessary, remove it.
- A stable timer text is better than an attention-grabbing flicker.
- If visual emphasis is needed, use a subtle color change or tiny badge pulse only on a dot/icon, not the whole block.

3. Avoid full rerender on timer tick
- If the timer uses setInterval / requestAnimationFrame and re-calls a full render function, refactor so only the timer text content updates.
- Keep current event handlers intact.
- Do not break Play / Replay / Stop behavior.

4. Respect accessibility
- No flashing effect on large surfaces.
- Respect `prefers-reduced-motion`.
- Keep text readable and stable.
- No rapid flashing.

5. Preserve current behavior
- Listening playback gating still works
